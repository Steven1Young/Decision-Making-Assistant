# V0.1 Spec - Personal Cognitive System

> V0.1 的目标不是功能完整，而是验证核心体验是否成立。

## 1. Version Goal

V0.1 的目标：

> 让用户每天愿意打开，并在混乱、迷茫、复盘和选择中得到清晰感。

V0.1 不追求成为完整平台。

它只需要证明一件事：

> 混乱输入可以被系统转化为更清楚的问题、更准确的本质判断和可维护的长期记忆。

## 2. Target User

V0 阶段只服务用户本人。

不考虑多人通用化，不优先考虑泛化产品能力。

所有设计默认围绕一个长期用户展开。

## 3. Priority Scenarios

V0.1 优先服务三个场景：

1. 情绪与迷茫整理
2. 学习与成长复盘
3. 职业 / 学业决策

这些场景的共同特点是：

- 问题往往不清楚
- 用户可能混杂了事实、感受、恐惧、推测和价值冲突
- 没有标准答案
- 直接给建议容易误导
- 真正价值在于澄清问题和提炼本质

## 4. First Core Entry

V0.1 第一核心入口：

> 我现在很乱，帮我梳理。

这个入口最贴近项目原点：

- 难办的问题
- 难以分析的事情
- 难以安放的自己

它可以自然覆盖迷茫整理、职业学业选择、阶段复盘和长期画像更新。

## 5. Core Loop

V0.1 的核心链路是：

```text
混乱输入
↓
澄清问题
↓
区分事实、感受、假设、担忧和价值冲突
↓
提炼本质
↓
形成初步观点
↓
必要时追问或挑战
↓
生成 Session Summary
↓
提出 Memory Update 建议
```

## 6. Core Capabilities

### 6.1 Problem Clarifier

系统不急着解决问题，而是先澄清问题。

需要完成：

- 复述系统理解到的用户状态
- 拆分事实、感受、假设、担忧、价值冲突
- 判断表面问题和真正问题是否一致
- 找出当前最关键的不确定性
- 必要时提出少量高质量追问

### 6.2 Independent Analysis

系统必须有自己的分析和观点。

但观点必须：

- 说明原因
- 说明证据
- 说明反面可能
- 说明不确定性
- 接受后续修正

### 6.3 Essence Judgment

每次重要对话都应尝试给出“本质判断”。

本质判断不是鸡汤，也不是标签，而是对问题结构的提炼。

示例：

> 这件事的本质不是你不知道怎么选，而是你还没有确定自己愿意为哪种路径付出代价。

或者：

> 这次迷茫的本质不是懒惰，而是目标体系暂时失效：你过去依赖外部评价推进自己，但现在需要建立内部判断标准。

### 6.4 Session Summary

每次重要对话结束后生成总结。

风格：战略备忘录 / 个人复盘。

不采用心理咨询记录风格。

建议结构：

```text
## Session Summary

### 表面问题
...

### 本质判断
...

### 已澄清的事实
...

### 仍未验证的假设
...

### 观察到的思维模式
...

### 长期趋势信号
...

### Memory Update 建议
...

### 下一步行动（如有必要）
...
```

下一步行动不是必填项。

如果本次对话更适合沉淀理解，而不是立刻行动，则不强行生成行动清单。

### 6.5 Memory Update Suggestion

V0.1 不应自动无脑写入长期记忆。

每次重要对话结束后，系统提出 Memory Update 建议：

- 建议新增什么
- 建议修改什么
- 建议不记录什么
- 为什么值得记录
- 置信度如何
- 是否需要用户确认

重要记忆必须经过用户确认。

### 6.6 Local-First Memory

V0.1 的长期记忆默认本地存储。

敏感内容默认不上传。

如果还没有实现完整隐私机制，应在产品和代码层面明确标注限制。

## 7. Non-Goals

V0.1 不做：

- 多用户系统
- 社交功能
- 完整心理咨询流程
- 医疗、法律、财务专业建议
- 自动投资或人生决策建议
- 复杂 Hermes / AI 维护 AI 流程
- 完整 Framework Library
- 过度自动化的 Memory 写入
- 面向公众的通用产品化

## 8. Minimum Data Model Draft

这是 V0.1 的概念性数据模型，具体实现后续确定。

### 8.1 User Profile

保存长期画像：

- long_term_goals
- values
- current_life_stage
- important_projects
- thinking_patterns
- emotional_patterns
- decision_preferences
- knowledge_background
- constraints

### 8.2 Memory Item

单条长期记忆建议字段：

- id
- type
- content
- evidence
- confidence
- sensitivity
- status: pending | confirmed | rejected | archived
- created_at
- updated_at
- review_at

### 8.3 Session Summary

单次对话总结建议字段：

- id
- date
- surface_problem
- essence_judgment
- clarified_facts
- unverified_assumptions
- observed_patterns
- long_term_signals
- memory_update_suggestions
- next_actions_optional

## 10. Version Acceptance Criteria

V0.1 可以被认为成立，当它满足：

- 用户可以输入混乱表达，并得到清晰的问题拆解
- 系统能区分事实、感受、假设和价值冲突
- 系统能给出本质判断，而不是只给建议
- 系统能形成有证据的独立观点
- 每次重要对话后能生成战略备忘录式 Session Summary
- 系统能提出 Memory Update 建议，并等待用户确认
- 用户感觉对话结束后更理解问题、更理解自己、更接近事实

## 11. Initial Roadmap

### V0.1

目标：每天愿意打开。

能力：

- 长期 Profile
- Problem Clarifier
- Session Summary
- 独立分析
- Memory Update 建议

### V0.5

目标：开始真正理解用户的长期变化。

能力：

- Goal Tracking
- Monthly Review
- Framework Library 初版
- 长期趋势识别
- Memory Review

### V1.0

目标：形成真正意义上的 Personal Cognitive System。

能力：

- 长期理解用户
- 发现重复模式
- 辅助复杂决策
- 沉淀认知成长
- 帮助用户形成越来越好的思考方式
